# 引言

欢迎光临！维基百科，这部免费的百科全书，是世界上最大和最受欢迎的参考网站。维基百科也是独一无二的：这部百科全书由每个人编写，任何人都可以阅读。

本书是为读者、当前编辑者、潜在贡献者和对维基百科感兴趣的任何其他人所写。本书描述了维基百科包含的写作类型，维基百科幕后是如何运作的，以及如何参与其中。

我们涵盖了参与维基百科的所有方面，从阅读网站到编辑文章，再到导航网站社区和治理。

维基百科基于一个*维基*，这是一种允许任何人轻松更改网站上页面的技术。如果你迫不及待地想在维基百科上工作，请访问[`en.wikipedia.org/`](http://en.wikipedia.org/)，立即开始改进这部百科全书！

如果你通过直接深入学习最快，或者如果你已经编辑过维基百科，那么你可以将这本书作为参考指南和技巧来源。但如果你是初学者，或者你想了解维基百科今天是如何运作的以及它是如何随着时间的推移而发展的，你应该从开头开始阅读。

# 本书内容概览

本书分为四个部分，从基础知识开始，逐步介绍维基百科的所有方面。

第一部分从读者的角度审视维基百科。第一章描述了维基百科包含的内容类型以及决定哪些内容被包含的基本政策。网站的历史在第二章中探讨；特别是，我们考察了维基百科如何将三个历史线索联合起来：百科全书、维基和免费软件。如何在网站上搜索和浏览（包括对侧边栏和主页的解释）在第三章中详细说明，而第四章则涵盖了单篇文章的结构和评估文章质量的方法。

第二部分转向编辑网站。第五章（hw-wkpd-wk_07.html "第五章。基本编辑"）中提供了如何编辑页面和介绍*维基语法*的基本解释；这一章构成了本部分后续章节的基础。第六章涵盖了如何开始新文章，并探讨了百科全书式的写作、研究和协作技巧，这些技巧对于处理任何文章都很有用。维护内容的正式流程，包括协作编辑驱动、清理项目、Wiki 项目和文章删除和提升流程，在第七章（hw-wkpd-wk_09.html "第七章。清理、项目和流程"）中有所涉及。第八章解释了文章是如何通过维基百科的分类结构、消歧义页面和重定向相互链接的，以及页面是如何通过合并、拆分和移动来维护的。更高级的语法，包括图像格式化、表格、模板和特殊字符，在第九章（hw-wkpd-wk_11.html "第九章。图像、模板和特殊字符"）中有所涉及。第十章逐步介绍了新创建文章的生命周期。

第三部分涵盖了维基百科复杂的社会层面。我们从如何注册成为编辑开始，包括设置账户、设置偏好和使用个人用户和讨论页面，在第十一章中，其中我们还描述了维基百科的管理员系统。维基百科的文化以及编辑之间的沟通方式（包括提问和讨论问题的论坛列表）在第十二章中有所涉及，维基百科的管理政策和这些政策的制定方式在第十三章中进行了描述。最后，第十四章讨论了争议解决过程是如何运作的，以及如何从一开始就避免分歧。

第四部分从英语维基百科转向覆盖其他维基媒体基金会项目，包括第十五章中其他语言的维基百科版本以及维基百科的姊妹项目，包括维基共享资源、维基词典、维基新闻、维基教科书、维基引言、维基源、维基物种和维基大学，在第十六章中。最后，我们讨论了维基媒体基金会（维基百科的母组织）以及如何在基金会层面协调工作，在第十七章中。

附录 A 包含有关在 GNU 自由文档许可下重用维基百科内容的信息以及重用示例（本书所许可的 GNU 自由文档许可本身，从 PREAMBLE 开始包括在内）。我们为在课堂上使用维基百科的教师提供了简要指南，见附录 B；附录 C 包含了在编辑摘要中常用术语的词汇表；附录 D 包含了网站上常用术语的词汇表；最后，附录 E 是书中引用的维基百科页面的索引。

我们描述的基本原则，用于阅读文章、编辑页面和与他人协作，将为在任何一个维基媒体维基项目上工作提供一个良好的基础。尽管维基媒体项目都共享相同的一般理念，但这里提到的任何具体政策或习俗可能不适用于英语维基百科之外。如果您对探索另一个项目感兴趣，请记住，每个维基网站代表一个独特的协作社区，可能有自己的规则。

# 进入之前您应该知道的内容

您不需要任何关于维基或编程的知识，就可以使用本书。但您应该知道以下方式如何访问互联网上的网页：

+   使用鼠标点击链接

+   直接在浏览器中输入页面地址（URL）

您还应该知道如何使用谷歌或其他搜索引擎在网络上进行搜索，并拥有一个活跃的电子邮件账户。

本书中对维基百科界面的所有引用都假设您正在使用个人电脑上的标准网络浏览器，已登录英语维基百科，并且使用默认（Monobook）皮肤且未进行修改。如果您未登录使用维基百科，则并非所有标签页和侧边栏链接都会显示，但大多数其他功能应该相同。

如果您打算在手机或个人数字助理（PDA）上查看维基百科，请访问 [`en.wap.wikipedia.org/`](http://en.wap.wikipedia.org/)。这个官方的移动版本是在 2007 年中引入的；有关非官方版本，请参阅 [[维基百科：WAP 访问]]。

# 使用本书

维基百科总是在不断变化。它是一个不断运动的网页集合，因此尽管我们已尽可能使本书保持最新，但具体细节在您阅读这些文字时可能已经改变。因此，我们经常引用维基百科上的页面。

在正文内，维基百科页面名称用双括号括起来，例如：[[帮助：编辑]]。您可以通过输入以下内容之一来访问该页面：

+   将标题 *帮助：编辑* 输入到维基百科的搜索框中（冒号后不要插入空格）

+   直接在浏览器地址栏中输入页面的 URL，例如，输入 [`en.wikipedia.org/wiki/Help:Editing`](http://en.wikipedia.org/wiki/Help:Editing)

### 注意

*所有英语维基百科的网址都以* [`en.wikipedia.org/wiki/`](http://en.wikipedia.org/wiki/) *开头，*双括号中的页面名始终指的是 URL 的最后一部分。因此 [[了不起的盖茨比]] 指的是* [`en.wikipedia.org/wiki/The_Great_Gatsby`](http://en.wikipedia.org/wiki/The_Great_Gatsby)。

在完整 URL 中，空格会自动转换为下划线；你不需要手动输入它们。

对于某些社区和政策页面，我们提供了以 *WP* 开头的快捷方式。维基百科的贡献者经常在他们讨论中使用这些缩写作为简写，例如：

```
[[Wikipedia:How to edit a page]], shortcut WP:EDIT
```

将快捷键输入维基百科的搜索框中，即可跳转到列出的较长页面标题。

每个章节的结尾，我们列出最重要的参考页面。在这里，你可以轻松访问特定主题的链接，这对于经验更丰富的维基百科用户来说也是一个参考。我们直接引用的每个维基百科页面都列在 附录 E 中。

# 我们理解维基百科的方法

我们提供的许多技术信息也可以在维基百科的帮助和项目页面上找到，尽管这些页面并不总是井然有序。在撰写这本书时，我们试图利用自己的经验来解释（并浓缩）数千页文档中的选择，花费时间阐明未成文的习俗，并指出最基本政策对新编辑的影响，以提供一个关于维基百科及其实际运作的视角。

尽管我们试图表达关于维基百科运作的共识观点，但本书中表达的观点仅代表作者的观点。这些观点不一定被其他编辑（包括被引用的编辑）或维基媒体基金会所共享。维基百科现在 *非常庞大*，每位编辑在网站上的体验都将是不同的。你在维基百科上的经历可能与我们所描述的有所不同。本书的作者在维基百科上共同拥有 11 年的工作经验和超过 15 万次的编辑，但我们仍然只直接体验了网站的一部分。在这本书中，我们主要讨论共同点：适用于整个项目的通用原则。

对维基百科做出绝对、确定的陈述是棘手的。即使是基本政策也可能改变——明天的维基百科可能就不同了。适应性和变化实际上是这个项目的标志。尽管如此，项目在宪法和政策方面的许多设置似乎在编辑社区中相对稳定和被理解；并且经过七年的存在，维基百科已经具有一定的成熟度。我们尽力在出版时做到尽可能准确，但如果你需要更确切的信息，请始终参考网站上的实时页面。
