# 第十章。文章的生命周期

到目前为止，我们从读者和编辑的角度广泛地考察了维基百科。当然，维基百科的读者可以随心所欲地来去，即使是最热情的维基百科编辑有时也会暂时离开电脑。但维基百科的*文章*总是每天在网站上。

那么，从那篇文章的角度看，事情看起来如何？

让我们跟随由爱迪这位编辑创建的阿蒂这篇文章。也许阿蒂的标题是[[Gingerbread cottage architecture]]，这是在第六章中之前使用的标题。

# 文章的诞生

爱迪在搜索框中输入`Gingerbread cottage architecture`。他发现这篇文章还不存在，从搜索页面跟随“开始新页面”链接，然后写下几句话。他点击了“保存页面”按钮：阿蒂诞生了。

当爱迪保存他的新文章时，它就会“上线”，可以通过搜索功能链接和发现。但同样重要的是，它的标题会立即显示在名为[[Special:Newpages]]的列表顶部。这个页面列出了最近创建的 5,000 篇文章。[[[Gingerbread cottage architecture]]]将在列表中下滑两三天，直到其他编辑——以及可能巡逻的维基百科管理员——审阅这些新文章。

在初步审查之后，可能出现多种结果。

# 删除

维基百科的文章是在一个敌对的环境中创建的，特别是那些只有几句话的简短文章——这些文章尤其危险。它们在维基百科的池塘中不过是蝌蚪。那些看起来不适合网站的新文章在审阅时经常被标记为删除；这是每天数百篇文章的命运，其中许多都是出于好意。

因此，爱迪应该密切关注他的文章。如果内容非常糟糕——如果包含涂鸦或是不符合百科全书风格，或者如果主题似乎没有充分引用以证明其重要性——可能会在文本开头添加一个*红色栏*模板，提名该文章进行删除。（有三种类型的删除提名，每种都使用不同的模板；参见第七章。）

爱迪不应自己移除删除模板，但他可以争议删除提名。例如，他可以通过在删除模板下方添加`{{hangon}}`模板到文章中，然后立即在文章的讨论页面上为自己的立场辩护，来争议快速删除提名。

如果一个文章被添加了删除标签并且没有被争议，阿蒂的未来就会变得暗淡，所以爱迪需要尽快了解任何删除提名。爱迪可以通过几种方式跟踪他文章的变化：

+   Eddie 可以通过勾选“监视此页面”将文章添加到他的监视列表中（这是为你创建的页面默认执行的）。Eddie 的监视列表将揭示文章的删除提名和其他编辑。

+   Eddie 可以检查他的用户讨论页面。每当一篇文章被提名删除时，其创建者应该通过创建者的讨论页面（在这种情况下，应在[[User talk:Eddie]]上留下消息）被告知。然而，这并不是万无一失的，因为并非所有编辑都会遵循这个习俗。

+   Eddie 可以在他的用户空间中保持一个*自夸列表*——一个指向他已创建的文章链接的列表。如果他把这个自夸列表放在单独的页面上（例如，[[User:Eddie/Articles I created]]），然后他可以从那个页面点击“相关更改”链接，以方便地查看最近编辑的列表。这个解决方案甚至比监视监视列表来关注你创建的文章还要好。

如果其他维基百科编辑认为文章内容足够好，Artie 将避免立即被删除。现在的工作是改进文章。

# 维护标记

当编辑们遇到需要关注的文章时，他们会标记这些文章以进行维护。许多这些*黄色条*模板可能会添加到文章的开头。例如，如果文章需要格式化工作或为了清晰度而重写，可能会添加`{{cleanup}}`标签；如果没有合适的参考文献，很可能会在文章上放置`{{unreferenced}}`标签。其他黄色条模板可能更技术化，例如，`{{film-fiction}}`：*这篇文章可能无法在事实和虚构之间做出明确的区分*。对写作标准的特定批评可能以*橙色条*模板的形式出现。例如，如果文章需要更多的维基链接，可能会添加`{{wikify}}`。Eddie 不应该把这些模板当作个人攻击——他正在得到对他工作的反馈，现在知道文章需要如何改进。对于 Eddie 来说，一个好的主意是尽可能地根据任何此类消息修复文章。

如果 Artie 仍然是一个草稿（换句话说，只是一个初步的处理，缺少一些基本内容），编辑可能会用`{{stub}}`模板标记他。但由于草稿会按类别积极排序（第八章中有所介绍。如果编辑提议合并，他或她会用*紫色栏*模板标记文章。

如果艾蒂被合并，那么艾迪创建的内容将被包含在一个包含姜饼屋建筑材料的更大文章中。艾蒂不会消失，但将变成一个谦卑的重定向页面。艾迪应该通过在相关文章讨论页面上讨论来反对任何仓促的合并或重定向提议。

# 讨论和内容标签

读者或编辑，可能是访问维基百科的非常规访客，可能会质疑文章的内容。这是真的吗？这是全部真相吗？这是有偏见的吗？可以提供特定断言的参考文献吗？这些观点可能会被添加到[[讨论:姜饼屋建筑]]；尽管在某些情况下，评论可能会被添加到[[用户讨论:Eddie]]——让我们希望是礼貌的。

这种输入是进一步改进阿蒂的清晰度和准确性的机会。评论也可能采取橙色条模板的形式。例如，模板`{{NPOV}}`表示有人认为文章未能保持中立（不符合中立观点政策）。添加该模板的人也应该添加评论说明其理由。提出内容问题的模板，如果不是完全自解释的，应该总是有讨论页面的评论来具体说明问题或偏见。没有这样的详细说明，爱迪可能会对需要修复的内容感到困惑。

如果爱迪还在关注这篇文章，他应该回应所有合理的查询，而不是感到烦恼。当然，简单地移除请求某种澄清的标签对文章并无好处，除非这个标签完全是不应得的。一个标签在文章上停留一段时间并不会造成真正的伤害，在维基百科上内容被重写是很正常的事情，即使提出的问题仅涉及写作风格的细微改进。

# 分类

读者可以轻松找到关于广泛主题（如美国）的文章，但更难找到关于较小或更具体主题的文章——尤其是如果他们不知道文章的确切标题。这些不太突出的文章通常是通过编辑搜索分类来找到的。

如果已经添加了正确的分类标签，编辑找到并编辑文章的机会就会提高。爱迪可能自己会这样做。如果你正在开始一篇新文章（或对一篇文章进行编辑），请注意类似文章被放置的分类。

即使早期分类并不完美，也没有关系。即使是一个大致的分类也能将文章置于专家可以应用适当子分类的位置。

# 机器人到来

偶尔是程序员的编辑可以编写软件来自动执行某些类型的程序编辑。这些程序被称为*机器人*，阿蒂在其一生中可能会被许多这样的机器人访问。一些机器人会根据*风格指南*进行拼写校正或小的格式更改，而其他机器人则通过关键词等方式分析新页面的内容，并将其记录在项目页面保留的各种列表中。（日志可以在反向链接中检测到。）机器人所做的任何编辑都会在文章的历史记录中清楚地显示出来，就像人类编辑所做的编辑一样；机器人只是另一种账户类型。（例如，机器人的用户名几乎总是表明它是一个机器人而不是人类，例如 Sinebot，当编辑未能这样做时，它会绕着讨论页面签名。）

如果阿蒂还没有被分类，阿蒂的第一个机器人编辑可能就是添加一个`{{未分类}}`标签。

### 注意

*看到机器人行为不当？您通常可以通过机器人的用户讨论页面联系到机器人的程序员。所以如果机器人用一些自动化编辑弄错了 Eddie 的文章，Eddie 可以撤销编辑，并通知机器人的所有者*。

# 进来的维基链接

在我们关于文章创建的全面审查第六章中，你了解到指向新文章的维基链接应该添加到相关文章中（这些文章本身可以通过搜索功能找到）。实际上，这些链接最好在新的文章存在之前就添加。

如果 Eddie 在创建文章方面有经验，他一开始就会考虑进来的维基链接。当点击侧边栏上的“链接到此页”链接时，Artie 创建后立即显示多少页面？Eddie 应该检查。对于 Artie 来说，成为一个孤儿并不那么有趣。如果 Eddie 不添加链接到他的文章，其他人可能会添加`{{orphan}}`黄色栏模板，将 Artie 放入[[Category:Orphaned articles]]。

如果没有链接到文章，可能有几种解释。例如，Artie the Article 所选的标题[[Gingerbread cottage architecture]]可能是不寻常的或拼写错误。也有可能该概念或文章标题实际上在其他文章中提到，但尚未被维基链接。在创建文章后，Eddie 仍然可以创建指向 Artie 的链接。创建维基链接也将（微妙地）引起编辑对文章的关注，因为维基化的页面可能已经在相关领域的编辑的观察列表中。Eddie 还应该从其他可能的标题创建到 Artie 的重定向。

**关键词分析的一个例子**

关于美国将军的[[Carter B. Magruder]]文章，在 2007 年 9 月创建后不久就被 Alex Bakharev 运行的机器人抓取。该机器人将文章添加到 22 个日志中，例如 *User:AlexNewArtBot/OhioLog, User:AlexNewArtBot/VirginiaLog, User:AlexNewArtBot/WWIILog, 和 User:AlexNewArtBot/ColdWarLog*。这些只是短期的新文章列表，保存在用户命名空间中，并通过分析文章文本创建。这些日志现在通常会被传递给相关的维基项目，类似于谷歌警报的风格。*User:AlexNewArtBot/OhioLog*中关于俄亥俄州的新文章列表并非由于文本中出现了*Ohio*，而是由于短语*Society of the Cincinnati*（指一个历史协会，而不是俄亥俄州的城市）中出现了*Cincinnati*。

因此，机器人显然有局限性：它们可能会错误地建议文章与某个主题领域相关，而且它们偶尔会犯人类不会犯的错误。当然，任何人都可以撤销无用的编辑，并在必要时从类别或日志中删除文章。

# 搜索引擎找到这篇文章

一旦创建了像 Artie 这样的文章，它就会被索引并通过维基百科内置的搜索找到。通用搜索引擎将在几周内或有时甚至更早——对于 Google 来说，可能是一两天内——找到 Artie。一旦被搜索引擎注册，Artie 在网上的知名度将大大提高。那些在网络上搜索的人，并不是专门寻找维基百科文章的人，将会开始阅读出现在他们搜索结果中的文章。这可能会对文章的质量产生好的或坏的影响。该主题的专家可能会对其做出贡献，但随机的外部人员也可能进行破坏。为了 Artie 的利益，我们希望他现在已经被一些留心的编辑列入了观察名单。

# 新的亲属

随着时间的推移，Artie 的内容将以两种方式向外传播：它将被逐字复制，并且可能被翻译成其他语言。

大约一个月内，非维基百科网站将抓取 Artie 的内容并全文重新发布。这些*镜像*只要尊重 GFDL 许可证，就是完全合法的。随着这些副本在网络上传播，Artie 在维基百科上的内容可以通过持续更新而保持相关性。

阿蒂的内容也可能被翻译，为其他语言的维基百科提供内容。这些新的维基文章可能会吸引进一步的编辑，其内容可能开始与原始文章有所不同。

一个注意事项：如果阿蒂含有错误，那么阿蒂的镜像和翻译也会有错误——即使错误在阿蒂中被修复，它仍然会留在镜像网站上（至少直到镜像网站从维基百科的最新版本刷新其内容）和翻译中（直到有人手动纠正它）。

# 获取图片

一幅图胜过千言万语。

爱迪可能会被诱惑在互联网上某个地方找到相关图片，上传到维基百科，并将其添加到他的文章中，但除非该图片是公有领域或 GFDL 许可的（而且大多数都不是），这种添加只会带来麻烦。非自由图片会被维基百科积极删除；它们的生命就像切花一样短暂——最多几天。

相反，爱迪可能自己拍照（或创建图形）。或者他可能在其他维基百科文章或维基媒体共享资源中搜索相关图片。

# 优秀文章

假设爱迪的文章避免了之前讨论的危险，阿蒂可能由熟练的编辑开发，他们知道如何逐步改进页面。除了常规编辑外，这些编辑可能会让阿蒂通过一系列结构化的改进流程，例如，正式的同侪评审。如果阿蒂被正式认定为“优秀文章”，就会打开更多的可能性，包括被纳入那个稀有的层次，称为“特色文章”。主页在召唤！

**文章质量**

只有大约 0.1%的文章符合特色文章的标准，所以这个寓言有点过于乐观，而且需要一个仙女教母来帮忙。一些有奉献精神的编辑旨在从头开始制作特色文章，但大多数文章逐渐达到这个质量标准。

其他编辑生产大量较短的片段。从百科全书的角度来看，这些方法相互补充，适合不同的编辑气质。真正重要的是，文章最终是集体和彻底撰写的。维基百科没有一种正确的写文章方式。

# 摘要

产生维基百科内容的整个系统可能看起来有些混乱或随机。它当然是有缺陷的。内容是从一个复杂但善意的开发过程中产生的，在这个过程中，可能前进一步，然后退一步。但维基百科提供了许多层级的审查和改进，即使没有一套单一的程序，最终维基百科吸引读者是因为其内容总体上非常实用。确实，维基百科在互联网上的足迹正在稳步增长。

# 第二部分结论

维基百科需要所有类型的文章。你应该写些什么呢？存在许多新文章的可能性，但维基百科现有的两百万篇文章中的大多数仍然需要改进。如果你想帮助撰写文章，你总是可以应用基本的维基化格式化（[第五章](hw-wkpd-wk_07.html "第五章。基本编辑"）），致力于清晰重写和引用事实（[第六章](hw-wkpd-wk_08.html "第六章。良好的写作和研究"）），执行清理任务（[第七章](hw-wkpd-wk_09.html "第七章。清理、项目和流程"）），整理超文本和分类问题（[第八章](hw-wkpd-wk_10.html "第八章。构建和修复维基百科的网络"）），甚至可能贡献图片或专家语法（[第九章](hw-wkpd-wk_11.html "第九章。图片、模板和特殊字符"））。

但我们关于为维基百科写作的核心建议是，对于一篇文章来说，以下四件事最为重要：

+   从一开始就*写得相当好且来源可靠*有助于避免可能的删除，并为一篇好文章打下基础。

+   *遵守基本内容政策*，包括中立性、可验证性和无原创研究。

+   *与现有的维基百科材料相匹配*，以便存在或可以创建进入的维基链接。

+   *关于一个对其他人也感兴趣的话题*，可能适合在 WikiProject 的范围内，这样其他人就会找到并开发它。

良好的编辑家通常可以发现维基百科中目前尚未充分开发的部分，在那里可以创建新文章。同样常见的是，许多文章存在，但它们状况不佳，需要大量的结构和链接工作。当你准备好处理这类工作时，你就掌握了这本书的这一部分。然后你可以认为自己是一位高级编辑。
