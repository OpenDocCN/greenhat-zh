# **引言**

在许多小型和成长型公司中，工程经理常常处于一个独特的位置，既要处理技术团队和其他高级经理，同时还要从首席执行官那里接受指导。开发经理往往只关注技术，尽管工作的非技术方面可能对公司成功的影响最大。随着公司的发展，曾经看似微不足道的问题可能会相应增长，最终演变成重大灾难。我写了《软件成长》这本书，为那些面对这些多样化挑战的新开发经理提供建议。

小型公司的开发经理的角色在许多方面都不同于大型稳定公司的相同角色。例如，小型公司的开发经理必须经常与开发者合作，以支持一个不成熟的产品。开发经理还必须与那些被成长型公司面临的挑战所吸引的个性鲜明的个人合作。但最重要的是，小型公司的开发经理必须具有广泛关注的焦点，包括员工、产品、流程、规划、技术和客户。

相比之下，大型公司通常支持多个现有产品，他们的流程通常定义良好且相对静态。大型公司的政策通常限制了开发经理可以使用工具、技术和方法的选择。经理的角色比小型公司中的相同角色更具体，范围也更窄。

《软件成长》为小型公司的发展经理提供了一本实用、动手操作的指南，这些公司已经度过了最初的生存阶段，并试图成长。它的目的是帮助经理提前预见问题，并在问题变得难以控制之前解决它们。这里描述的技术对于生产软件销售或软件即服务的小公司很有用；它们并不针对软件咨询业务。《软件成长》提供了普遍的建议、具体的解决方案以及详细的模板和电子表格，以帮助开发经理将一般概念转化为实际行动。

由于本书的范围很广，它采用了规定性的风格而不是辩论性的风格——也就是说，许多建议并没有通过详尽的论证来说明为什么这些技术效果良好。尽管这些信息会大大增加本书的范围，但它会使本书的可读性降低。

为了方便，我在整本书中使用“开发经理”和“开发管理”这两个术语来描述**顶级**软件/工程经理——无论具体的职位名称是首席技术官（CTO）、工程副总裁、工程总监还是高级工程经理。这个人管理软件工程师，但他或她也可能管理质量保证、文档和项目管理团队。尽管本书的目标受众是负责所有开发工作的人，但非技术经理也会对这里描述的问题和解决方案感兴趣。

## 电子表格

电子表格示例贯穿全书，用于说明收集、分析和显示信息以解决特定类型问题的技术。每个示例都在文本中进行了说明和描述，并可以从[`www.nostarch.com/growingsoftware.htm`](http://www.nostarch.com/growingsoftware.htm)下载，用于 Microsoft Excel 或 OpenOffice.org Calc，根据每个程序的小差异进行调整。这些示例的主要目的是教授如何使用简单的电子表格分析和解决潜在问题。

虽然可以直接使用电子表格，但许多问题将需要您自定义基本电子表格布局。您可以通过将所有字段输入到电子表格程序中来重新创建电子表格。箭头指向特定的单元格，以指示可以输入的公式，描述位于示例的右侧或下方。根据需要，按照电子表格附带的说明复制公式到行或列。![图 1。此示例电子表格说明了单元格指令。](img/pr02.html#this_sample_spreadsheet_illustrates_cell "图 1。此示例电子表格说明了单元格指令。")展示了这些约定。

输入公式时，请注意美元符号 ($) 字符，它会影响公式在从一个单元格复制到另一个单元格时读取数据的方式。如果不包括美元符号字符，复制公式后可能会导致计算错误。

![此示例电子表格说明了单元格指令。](img/tagoreillycom20090324nostarchimages266057.png.jpg)

**图 1。此示例电子表格说明了单元格指令。**

正确的格式对于电子表格示例也很重要。默认的格式为“常规”，它不会以最合适的方式显示所有值。格式从示例中隐含：*日期*应使用日期格式；*货币*金额应使用货币格式，小数点后保留零位数字；*数字*应使用数字格式，四舍五入的选择基于感兴趣数字的位数。

**Excel 和 OpenOffice.org Calc 之间的主要差异**

+   OpenOffice.org Calc 使用分号 (;) 来分隔公式中的字段，而 Microsoft Excel 使用逗号 (,). 本书使用逗号来分隔所有公式中的字段。如果您在 Calc 中工作，请使用分号。例如，Excel 公式 *=WORKDAY(B3, C3)* 在 Calc 中的样子是这样的：*=WORKDAY(B3; C3)*。

+   第二个重要的区别是用于引用其他工作表中单元格的**跨工作表引用**。在 Excel 中，跨工作表引用是一个感叹号 (!)，而在 Calc 中是一个句号 (。例如，引用 *Eng* 工作表的公式在 Excel 中显示为 *=Eng!H3*，但在 Calc 中显示为 *=Eng.H3*。
